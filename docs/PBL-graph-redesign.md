# PBL 图谱重新设计：环形知识路径

> **状态：已实现** — 此设计已在 v3.0.0 (2026-04-21) 中完成实现。当前代码已反映此文档中描述的所有改动。
> 此文档作为设计决策的历史记录保留。当前实现详情请参阅 [FEATURES.md](FEATURES.md) 和 [ARCHITECTURE.md](ARCHITECTURE.md)。

## 参考图分析

参考图的核心特征：
1. 主路径是一条从左到右的项目序列（水平主线）
2. 分支项目从主路径分出，但最终**汇回主路径**
3. 每个项目下方挂着一串**知识点链条**，知识点之间有先后顺序，形成**环形路径**——从项目出发，经过知识点链，最终指回项目本身
4. **没有共享知识点**——每个项目有自己专属的知识链

## 关键设计规则

### 1. 主路径
- 4-8 个项目从左到右，用 **NEXT_STEP** 连接
- 难度递增：beginner → intermediate → advanced

### 2. 分支路径
- 从主路径的某个项目分出，用 **NEXT_STEP** 连接（不是单独的边类型）
- 分支项目最后通过 **NEXT_STEP** 汇回主路径的后续项目
- 结构：主项目 → 分支项目 → 汇回主项目（仍在主路径上）

### 3. 知识环形路径
- 每个项目有自己专属的知识点链条（2-5个知识点）
- 从项目出发：**REQUIRES** 项目→第一个知识点
- 知识点之间：**PREREQUISITE_OF** 前一个→后一个
- 最后一条知识指回项目：**REQUIRES** 最后一个知识点→项目（形成环形）
- **没有共享知识点**——每个知识点只属于一个项目

### 4. 边类型（仍然只有 3 种）

| 边类型 | 方向 | 颜色 | 含义 |
|--------|------|------|------|
| NEXT_STEP | 项目→项目 | 黑色粗线 | 项目路径顺序（主路径+分支） |
| REQUIRES | 项目→知识点 或 知识点→项目 | 紫色 | 项目需要知识 / 知识链最终指回项目 |
| PREREQUISITE_OF | 知识点→知识点 | 橙色 | 知识的学习先后顺序 |

### 5. 与当前代码的差异

| 维度 | 当前 | 新设计 |
|------|------|--------|
| 知识共享 | 允许多个项目共享知识点 | 禁止，每个知识只属于一个项目 |
| 知识路径形状 | 项目→知识点（单向扇形） | 项目→知识点1→知识点2→...→项目（环形） |
| REQUIRES 方向 | 只从 项目→知识点 | 项目→知识点 **和** 知识点→项目（环形闭合） |
| 分支项目 | 独立于主路径 | 从主路径分出并汇回主路径 |
| knowledge_sharing | 存在 | 不存在（每个知识只属一个项目） |

## 需要改动的代码

### 后端 `graph_extractor.py`

1. **Prompt 更新**：
   - 明确说明：每个项目的知识点形成环形路径，从项目出发最终回到项目
   - REQUIRES 边支持双向：项目→知识点（起点），知识点→项目（终点）
   - 禁止共享知识点
   - 分支项目用 NEXT_STEP 连接，从主项目分出并汇回主项目

2. **`_validate_and_process`**：不需要大改，REQUIRES 本身不限制方向

3. **`_compute_pbl_structure`**：
   - 移除 `knowledge_sharing`（不再有共享知识）
   - `project_knowledge_subgraphs` 改为记录每个项目的知识环形链
   - 分支项目识别：不是主路径上的项目，但通过 NEXT_STEP 与主路径相连

### 后端 `neo4j_operations.py`

1. **`_compute_pbl_learning_path`**：
   - 移除 `knowledge_sharing` 计算
   - 识别分支路径：NEXT_STEP 链中不在 main_path 上的项目
   - 每个项目的知识子图按环形路径计算

### 前端 `Process.vue`

1. **学习路径布局**：
   - 项目主路径水平排列在中间
   - 分支项目在主路径上方/下方
   - 知识环形路径：每个项目下方垂直排列知识点链，最后一条知识画弧线指回项目

2. **侧边栏**：
   - 移除"共享知识"部分
   - 每个项目下显示知识链（有序列表）

3. **环形边的渲染**：
   - 知识链的最后一条 REQUIRES 边（知识点→项目）需要画一条弧线回到项目节点
